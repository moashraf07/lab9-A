# lab9-A
The Prompt:"I need to build a monitoring system for a Node.js application running on Kubernetes using Prometheus and Grafana. Please provide the following:1.Node.js Application: Create a simple Express.js app that uses the prom-client library to expose default system metrics and custom HTTP request duration metrics at a /metrics endpoint.2.Dockerization: Provide a Dockerfile to containerize this application.3.Kubernetes Manifests:•A Deployment file with 2 replicas.•A Service file (NodePort) that includes a named port (e.g., name: http) so it can be discovered by Prometheus.•A ServiceMonitor custom resource that tells Prometheus to scrape the /metrics endpoint every 15 seconds.4.Monitoring Setup: Explain the Helm commands needed to install the kube-prometheus-stack.5.Verification: Provide the commands to port-forward the Prometheus and Grafana services to access their dashboards locally, and explain how to retrieve the default Grafana admin password from Kubernetes secrets.6.Load Testing: Provide a simple script (PowerShell or Bash) to generate traffic to the app to see the metrics change in real-time."Why this prompt works:•Specific Libraries: It mentions prom-client, which is the industry standard for Node.js.•K8s Best Practices: It specifically asks for a "named port" and a "ServiceMonitor," which are the most common points of failure in this lab.•End-to-End: It covers everything from the code to the visualization and testing.You can just copy and paste this whenever you need to recreate the project!
